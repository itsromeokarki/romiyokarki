Unintended Solution:

While creating the CTF , I made a regex mistake in admin XSS BOT which created this unintended solution , Luckily it wasn't abused to solve this challenge :)

Regex for requested URL :

^http://34.72.196.228:5051?.*

I wanted to make sure that @ redirection wont work so i made sure “?” should be included in XSS URL :

https://google.com?@google.com -> 200 OK 
https://google.com@yahoo.com -> 302 Redirection -> Yahoo.com

To read more about on this redirection: https://security.stackexchange.com/questions/252669/open-redirect-adding-symbol

But in regex , “?” means it can either contain the preciding character or not .

Example: colou?r matches both color and colour.

Which means now XSS bot will allow this URL :

http://34.72.196.228:5051@google.com -> 302

Now test it using own hosted webbook service:

http://34.72.196.228:5051@cgwkncc3qew31709fon7gda42v8mwkk9.oastify.com
